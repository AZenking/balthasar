# Feature 规约: 设置页与账户管理

**Feature 分支**: `010-settings-ui`

**创建日期**: 2026-07-07

**状态**: Draft

**输入**: `docs/MVP.md` "/settings" 页面 + "能创建账户" 完成标准,基于 002-account 后端 API

## 概述

本 feature 实现 `/settings` 页面的完整前端 —— 用户管理账户 (新建/编辑/归档/取消归档) 和退出登录。

后端 002-account 已提供账户 CRUD API,本 feature 是其前端 UI。当前 /settings 页面仅有登出按钮,缺少账户管理界面。

本 feature **不新增后端 API**,仅实现前端页面 + 组件。

## Clarifications

### Session 2026-07-07

- Q: 010 spec 的 edge case 写"编辑已归档账户: 允许编辑",但 002-account 的 `update` API 显式拒绝编辑已归档账户 (FR-011: CONFLICT "已归档账户不可编辑,请先取消归档")。010 应服从 002 还是增强 002? → A: 服从 002 — 已归档账户不可编辑,用户需先取消归档才能编辑 (不改后端,与"不新增后端 API"约束一致)。
- Q: 010 spec 的 US3 未明确编辑表单包含哪些字段,而 002-account 的 `update` API 仅接受 `name` 和 `currency` (initialBalance 不可编辑,002 SC-007)。编辑表单应包含哪些字段? → A: 仅名称 + 币种 (匹配 002 的 `update` API,不包含初始余额字段)。
- Q: 010 spec 的 FR-001 和 US2 写"每行显示名称、币种、余额",但 002 的 `account.list` 只返回 `initialBalance` (无 computed balance)。列表里的"余额"应显示什么? → A: 显示 `initialBalance`,标注为"初始余额" (诚实标注,不含交易汇总;当前余额属 Dashboard 职责,不在本 feature 范围)。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 创建账户 (Priority: P1)

已登录用户在设置页点"新建账户"按钮,展开内联表单,填写名称 (如"招商银行卡")、选择币种 (默认人民币)、输入初始余额,提交后账户出现在列表中。

**为何此优先级**: 没有账户就无法记账。MVP 完成标准 "能创建账户" 的直接体现。

**Acceptance Scenarios**:

1. **Given** 用户已登录, **When** 点"新建账户"展开表单,填写名称 + 默认币种 + 初始余额, **Then** 提交后账户创建,列表新增一行。
2. **Given** 名称为空, **When** 提交, **Then** 显示字段错误"请输入账户名称",不创建。
3. **Given** 名称超过 50 字, **When** 提交, **Then** 显示字段错误。
4. **Given** 初始余额输入非数字, **When** 提交, **Then** 显示字段错误。
5. **Given** 未登录, **When** 访问 /settings, **Then** 重定向 /login。

---

### User Story 2 - 查看账户列表 (Priority: P1)

已登录用户在设置页看到所有账户列表,活跃账户在前,已归档账户在后 (灰色标注),每行显示名称、币种、初始余额。

**为何此优先级**: 列表是用户进入设置页的第一眼,也是确认账户已创建的反馈。

**Acceptance Scenarios**:

1. **Given** 用户有 3 个活跃账户, **When** 打开 /settings, **Then** 显示 3 行账户,含名称+币种+初始余额。
2. **Given** 用户有 1 个已归档账户, **When** 打开 /settings, **Then** 已归档账户显示在活跃账户下方,有"已归档"标记。
3. **Given** 无任何账户, **When** 打开 /settings, **Then** 显示"暂无账户,请先创建"。
4. **Given** 数据加载中, **When** 页面渲染, **Then** 显示骨架屏。

---

### User Story 3 - 编辑账户 (Priority: P2)

用户点击某账户行的"编辑"按钮,展开内联表单预填当前名称和币种 (不含初始余额,该字段不可编辑),修改后提交,列表刷新显示新数据。

**为何此优先级**: P2 —— 用户输错账户名或币种需要纠正,但不阻断核心记账流程。

**Acceptance Scenarios**:

1. **Given** 账户列表, **When** 点编辑按钮, **Then** 该行展开编辑表单,预填名称+币种 (不含初始余额)。
2. **Given** 编辑表单, **When** 修改名称 + 提交, **Then** 列表刷新,显示新名称。
3. **Given** 编辑表单, **When** 清空名称 + 提交, **Then** 显示字段错误,不保存。
4. **Given** 编辑表单, **When** 点取消, **Then** 表单收起,不修改。

---

### User Story 4 - 归档/取消归档账户 (Priority: P2)

用户点击某账户行的"归档"按钮,确认后账户标为已归档,移到列表下方。已归档账户可点"取消归档"恢复。

**为何此优先级**: P2 —— 用户清理不再使用的账户,保持列表整洁,但不影响已有交易。

**Acceptance Scenarios**:

1. **Given** 活跃账户, **When** 点归档按钮, **Then** 弹出确认"确认归档?此操作不影响已有交易"。
2. **Given** 确认对话框, **When** 点确认, **Then** 账户标为已归档,移到列表下方,显示"已归档"标记。
3. **Given** 确认对话框, **When** 点取消, **Then** 不归档。
4. **Given** 已归档账户, **When** 点取消归档, **Then** 账户恢复为活跃,移到列表上方。

---

### User Story 5 - 登出 (Priority: P3)

用户在设置页底部点"登出"按钮,退出登录,返回登录页。

**为何此优先级**: P3 —— 功能已存在,本 feature 保留不改动。

**Acceptance Scenarios**:

1. **Given** 用户已登录, **When** 点登出按钮, **Then** 清除会话,跳转 /login。
2. **Given** 登出后, **When** 访问 /settings, **Then** 重定向 /login。

---

### Edge Cases

- 归档最后一个活跃账户后,用户在记账表单会看到"请先创建账户"提示 (008 已处理)。
- 归档有交易的账户: 交易保留,账户引用不变,列表中账户标为"已归档"。
- 编辑已归档账户: 不允许 (与 002 后端 FR-011 一致)。已归档账户行不显示编辑按钮,用户需先取消归档才能编辑。
- 创建同名账户: 允许 (名称不唯一),用户自行区分。
- 币种选择: 限定为系统支持的币种列表 (人民币/美元/欧元/日元/港币/英镑/澳元/加元/新加坡元)。
- 初始余额为 0: 允许,默认值。
- 网络断开: 创建/编辑/归档失败,显示"网络错误,请重试"。
- 并发: 两人同时编辑同一账户 → 后提交者覆盖 (LWW,与 002 后端一致)。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在 `/settings` 页面显示账户列表,活跃账户在前,已归档账户在后,每行含名称、币种、初始余额。
- **FR-002**: 系统 MUST 提供"新建账户"按钮,点击后展开内联表单 (名称、币种、初始余额)。
- **FR-003**: 创建账户时 MUST 验证: 名称非空且 ≤ 50 字,初始余额为有效数字。
- **FR-004**: 币种 MUST 提供下拉选择,默认"人民币",选项为系统支持的全部币种。
- **FR-005**: 初始余额 MUST 以元为单位输入,允许小数 (最多 2 位),默认 0。
- **FR-006**: 创建成功后 MUST 在列表中新增该账户行,表单收起。
- **FR-007**: 每个活跃账户行 MUST 提供"编辑"和"归档"操作按钮。
- **FR-008**: 编辑表单 MUST 预填当前名称和币种,修改后提交更新该账户。编辑表单仅含名称和币种两个字段 (初始余额不可编辑,与 002 后端 `update` API 一致)。
- **FR-009**: 编辑表单 MUST 提供"取消"按钮,放弃修改并收起。
- **FR-010**: 归档操作 MUST 弹出确认提示,确认后该账户标为已归档。
- **FR-011**: 已归档账户行 MUST 显示"已归档"标记,提供"取消归档"按钮。
- **FR-012**: 归档/取消归档后列表 MUST 立即更新 (账户位置移动)。
- **FR-013**: 无账户时 MUST 显示"暂无账户,请先创建"空状态。
- **FR-014**: 数据加载中 MUST 显示骨架屏。
- **FR-015**: 系统 MUST 在 /settings 页面底部提供"登出"按钮,点击后清除会话并跳转 /login。
- **FR-016**: 所有操作 MUST Mobile-First 适配。
- **FR-017**: 创建/编辑/归档失败时 MUST 显示错误提示。

### Key Entities

无新增。复用 002-account 的 Account 实体 (id, name, currency, initialBalance, archivedAt, createdAt, updatedAt)。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 设置页首次加载 (含账户列表) ≤ 2 秒。
- **SC-002**: 创建账户 (从点按钮到列表更新) ≤ 30 秒。
- **SC-003**: 页面在 iPhone SE (375px) 上无横向滚动。
- **SC-004**: 归档账户后列表立即更新 (该行移到已归档区域)。
- **SC-005**: 登出后立即跳转 /login (≤ 1 秒)。

## Assumptions

- 后端 002-account API 全部就绪 (create, list, update, archive, unarchive)。
- 账户为家庭级共享 (非成员私有),通过 familyId 隔离。
- 币种列表: 人民币(CNY)、美元(USD)、欧元(EUR)、日元(JPY)、港币(HKD)、英镑(GBP)、澳元(AUD)、加元(CAD)、新加坡元(SGD)。
- 初始余额以元为单位输入,存储为分 (×100)。
- 归档而非删除: 保留交易引用完整性,已归档账户的交易不受影响。
- 创建/编辑用内联表单 (不新建独立页面,与 009 筛选区折叠模式一致)。
- 归档确认用 `window.confirm` (MVP 简化,与 009 删除模式一致;V2 评估 shadcn Dialog)。
- 登出功能已存在 (当前 /settings 页面),本 feature 保留不动。
- 账户列表不实现排序 (按创建时间倒序,与 002 后端一致)。
- 不实现账户删除 (用归档替代,保留交易引用)。
- 不实现账户间转账 (V2 范围)。
- 不实现密码修改/个人信息编辑 (不在 MVP 范围)。
