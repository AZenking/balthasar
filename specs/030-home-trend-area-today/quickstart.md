# Quickstart: 首页趋势图改面积平滑 + 本日支出

**Feature**: 030-home-trend-area-today | **Date**: 2026-07-17 | **Spec**: [spec.md](./spec.md)

> Phase 1 output。端到端验证指南,证明三项变更(本日支出 + 本周趋势窗口 + 平滑面积)可运行。完整实现细节见 `tasks.md`(由 `/speckit-tasks` 生成)。

## 前置条件

- 已启动 PostgreSQL(testcontainers 或本地 `docker compose up`)。
- 已执行 `pnpm db:push`(或既有迁移),`transaction` / `budget` / `account` / `category` 表就绪。
- 027-mobile-home-revamp 已落地(本 feature 是其增量)。
- 环境变量与 Better-Auth 已配置(能登录、有默认 family)。

## 验证场景

### 场景 1:本日支出并入 hero 卡(主从层级)

**目标**:验证 FR-001/FR-002/FR-003(US1)。

1. 登录后进入首页 `/dashboard`。
2. 在今日插入一笔 `-¥50.00` 餐饮支出(经记一笔或直接 SQL)。
3. 重新加载首页。

**期望**:
- hero 卡左侧"本月支出"为**主大数字**(显示 `¥1,xxx.xx`,含今日 ¥50)。
- hero 卡**右侧**"本日支出"为次级较小数字,显示 `¥50.00`。
- 两数字在同一行/同卡内并排,移动端窄屏(375px)不溢出、不换行错位。

**边界**:
- 今日再插入一笔 `+¥30.00` 退款(`type='expense'`,正 amount)→ 本日支出变 `¥20.00`(退款冲减,FR-002)。
- 删除今日所有交易 → 本日支出显示 `¥0.00`(非 `-`、非空,FR-002)。

### 场景 2:趋势图窗口 = 本周(不随月份)

**目标**:验证 FR-004/FR-005(US2)。

1. 登录进入 `/dashboard`(确认今日所在本周)。
2. 记录趋势图 X 轴 7 个日期桶 = 本周周一..周日。
3. 用顶部年月箭头切换到**上月**。

**期望**:
- hero 卡"本月支出"变为上月数据。
- 趋势图 X 轴**仍显示本周**周一..周日 7 桶(与步骤 2 完全一致),不随月份变化。
- 今日之后的本周未来日(如今日周三,则周四..周日)显示为 0(补零,FR-005)。

**边界**:
- 切回当前月 → 趋势图仍是本周(窗口与 month 解耦)。

### 场景 3:平滑折线 + 渐变面积填充

**目标**:验证 FR-006/FR-007(US3)。

1. 在本周内插入若干非零支出(让趋势有 2+ 个非零点)。
2. 重新加载首页,观察趋势图。

**期望**:
- 折线相邻点之间为**平滑曲线**(目视无锐利折角)。
- 折线下方有**垂直渐变面积**:顶部为支出红(`--danger`)较高不透明度、底部贴 X 轴处透明。
- 折线**不跌破 0**(monotone 插值保证,非负数据点之间无下穿)。

**边界**:
- 本周全 0(无任何本周交易)→ 折线贴 X 轴,面积近乎不可见,图表骨架不破坏(US3 acceptance 3)。

### 场景 4:环比徽标已移除 + 隐私遮蔽

**目标**:验证 FR-008/FR-009(SC-005/SC-007)。

1. 重新加载首页,观察趋势卡标题区。

**期望**:
- 标题区只剩"本周支出趋势"标题,**无**"较上月 ↑X%"百分比徽标(FR-009 / SC-007)。

2. 点击顶部隐私开关(开启隐私模式)。

**期望**:
- 本日支出、本月支出金额均显示 `***`(自动遮蔽)。
- 趋势图 Y 轴刻度显示 `••`,Tooltip 金额显示 `***`。
- 趋势图**形状保留**(平滑折线 + 渐变面积可见,只遮蔽金额,FR-008)。

### 场景 5:降级(dayExpense 查询失败)

**目标**:验证 FR-003(US1 acceptance 5)。

1. 临时让 `dayExpense` 子查询失败(如 mock `getDailyTrend` 今日窗口抛错,或在测试环境断开特定路径)。
2. 重新加载首页。

**期望**:
- 本日支出区域显示 `¥--.--`(降级占位)。
- 本月支出、趋势图、最近账单等其它区块**正常渲染**,不受影响(对齐 027 SC-008)。
