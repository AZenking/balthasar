# Feature Specification: 首页趋势图改面积平滑 + 本日支出

**Feature Branch**: `030-home-trend-area-today`

**Created**: 2026-07-17

**Status**: Draft

**Input**: User description: "折线图 + 渐变面积填充; 其中折线图要求平滑;并且首页只展示最近一周;首页在本月支出 右侧增加本日支出"

## User Scenarios & Testing *(mandatory)*

本 spec 是对 027-mobile-home-revamp 已落地首页的三项增量调整:

1. 趋势图从纯折线 → **平滑折线 + 渐变面积填充**(视觉强化趋势走向)。
2. 趋势图时间窗从"本月每日"改为**本周(周一..周日,UTC)**——窗口固定、不随月份导航变化(见下方 Clarifications Q2)。
3. 首页 hero 卡在本月支出**右侧**并排新增**本日支出**金额,语义为主从层级:本月支出保持主大数字、本日支出为次级(见 Clarifications Q1)。

三项均为纯 UI/数据展示调整,不触碰记账、分类、账户等领域不变量,属 MVP 范围内(宪章原则一)。`recharts` 已在依赖中(026 引入),`getDailyTrend`/`getMonthSummary` 已存在;本 spec 重用现有聚合能力,只调整窗口参数、视觉映射、hero 卡布局。

## Clarifications

### Session 2026-07-17

- Q: 「本日支出」在 hero 卡上相对「本月支出」的视觉地位与排布? → A: **主从层级**——本月支出保持主大数字(027 FR-001 语义不变),本日支出为本月支出右侧的次级较小数字;移动端不溢出、不换行。
- Q: 切换到历史月时,趋势图「最近 7 天/一周」窗口锚定到哪几天? → A: **窗口固定为"本周"(当前周一..周日,UTC),不随月份导航变化**。月份切换只影响 hero 本月支出、分类、预算等月维度区块;趋势图始终显示本周这 7 天。
- Q: 今天在本周中间时(如周三),本周周四..周日未来日怎么处理? → A: **补零,7 桶固定**。周一..周日始终 7 个桶,未来日显示为 0(与 027 既有补零纪律一致,图表骨架固定)。
- Q: 趋势卡现有的「较上月 ↑X%」环比徽标怎么处理? → A: **移除环比徽标**。窗口改为本周后,"较上月"语义不再成立;趋势卡标题区只保留标题(可改为"本周支出趋势")。
- Q: 「本周」与「今天」的日期边界与刷新策略? → A: **UTC 边界 + 拉取时刷新**。本周 = 周一..周日 全 UTC;今天 = UTC 日。打开页面/重新查询时取一次,跨 UTC 午夜/跨周一**不自动刷新**(下次查询才更新)。与 027 既有 UTC 纪律一致,不引入定时器(符合宪章六 YAGNI)。

### User Story 1 - 本日支出并入 hero 卡 (Priority: P1)

用户打开首页,在原本只显示"本月支出"的 hero 主数字卡上,**右侧**并排出现"本日支出"金额。两列共享同一张卡片,语义上"日 ←→ 月"对照。**视觉层级为主从**:本月支出保持主大数字(027 FR-001 既有语义不变),本日支出为本月支出右侧的次级较小数字(足够可读但视觉权重低于主数字)。本日支出 = 当前 UTC 日所有 `type='expense'` 交易金额的 `ABS` 之和(含退款冲减,与 027 R9 type-driven 口径一致);无当日交易时显示 `¥0.00`。

**Why this priority**: 用户反馈的核心诉求是"每天花多少要一眼看见"。本月支出是周期视角,本日支出是即时反馈;并排展示能在一屏内同时给出"今天"与"本月"两个锚点。这是改动最小、感知最强的一项,P1。

**Independent Test**: 在只有当日一笔 -¥50 餐饮支出的场景下,hero 卡左侧"本月支出"显示 `¥50.00`(主大数字),右侧"本日支出"显示 `¥50.00`(次级);切换日期到次日(无交易),本日支出变 `¥0.00`,本月支出仍含昨天的 ¥50。

**Acceptance Scenarios**:

1. **Given** 今日已有 ¥50 支出且本月累计 ¥1200, **When** 打开首页, **Then** hero 卡左侧"本月支出"为主大数字显示 `¥1,200.00`,右侧"本日支出"为次级数字显示 `¥50.00`,两者在同一行/同卡内并排、移动端不溢出。
2. **Given** 今日无任何支出, **When** 打开首页, **Then** "本日支出"显示 `¥0.00`,不显示空态、不显示 `-`。
3. **Given** 今日有一笔 ¥100 支出和一笔 ¥30 退款(`type='expense'` 正金额), **When** 打开首页, **Then** "本日支出"显示 `¥70.00`(净支出,退款冲减,与 027 R9 口径一致)。
4. **Given** 隐私模式开启(027 FR-008), **When** 打开首页, **Then** "本日支出"金额与"本月支出"一样被遮蔽(`***`),遮蔽态视觉一致。
5. **Given** 本日支出加载失败(子查询超时/抛错), **When** 打开首页, **Then** 本日支出区域显示降级态(如 `¥--.--)`,不阻塞本月支出与趋势图展示(对齐 027 budget/assets 降级纪律 SC-008)。

### User Story 2 - 趋势图窗口改为本周(周一..周日) (Priority: P2)

趋势图时间窗从"本月每日(1 日..今天/月末)"改为**本周**(当前 UTC 周一 00:00..周日 23:59 +1ms,共 7 个日桶)。每日金额 = 该 UTC 日 `type='expense'` 的 `ABS` 之和。**窗口固定为本周,不随月份导航变化**:用户在首页用年月箭头切换到历史月时,hero 本月支出、分类、预算等月维度区块随之变化,但趋势图始终显示本周这 7 天(本周是即时反馈维度,与所选历史月份无关)。今天若在本周中间(如周三),本周周四..周日等未来日按 0 补齐(7 桶固定,图表骨架稳定)。

**Why this priority**: 这是三项里改动面居中、但对图表信噪比提升最大的一项。本月每日线条在月中常有长段 0 值(月初无交易),趋势被压平;本周 7 天聚焦近期波动,趋势更易读。窗口固定为本周(而非跟随月份)避免"历史月份的本周"语义歧义,实现也更简单。P2。

**Independent Test**: 切换到上月,hero 卡"本月支出"变为上月数据,但趋势图 X 轴仍显示本周周一..周日 7 桶(今天所在周),不随月份变化;今天之后的未来日桶为 0。

**Acceptance Scenarios**:

1. **Given** 当前为本周周三, **When** 打开首页, **Then** 趋势图 X 轴显示本周周一..周日共 7 个日期桶,每桶为该日支出净额;周四..周日(未来)为 0。
2. **Given** 切换到历史月(如上月), **When** 趋势图重渲染, **Then** 趋势图窗口**仍为本周**(当前周一周日),不随所选月份变化;hero 本月支出/分类/预算变为历史月数据,但趋势图保持本周。
3. **Given** 本周某日无交易, **When** 渲染, **Then** 该日桶为 0(不跳过、不折线断开),与 027 daily 补零纪律一致。
4. **Given** 隐私模式开启, **When** 渲染, **Then** Y 轴刻度按 027 FR-008 遮蔽为 `••`,趋势形状保留。
5. **Given** 跨 UTC 周一打开页面后停留在页面内(不刷新), **When** 进入新的一周, **Then** 趋势图**不自动刷新**(下次查询/重拉才更新到新本周),符合"拉取时刷新"约定。

### User Story 3 - 平滑折线 + 渐变面积填充 (Priority: P3)

趋势图的折线视觉强化为两层:

- **平滑折线**:相邻数据点之间以平滑曲线连接(而非折角),体现"趋势"而非"逐点"。
- **渐变面积填充**:折线下方填充从折线色(支出红,`--danger`)到透明的垂直渐变,强化趋势走向的视觉重量。

**Why this priority**: 纯视觉增强,不改数据语义。放在 US1/US2 之后,因为面积填充在"本周 7 天"的短窗口上观感最佳(本月长窗口下面积易显拥挤)。P3。

**Independent Test**: 给趋势图喂 7 个非零数据点,目视确认折线为平滑曲线(无折角)、折线下方有从顶部不透明到底部透明的红色渐变填充;切隐私模式,面积/折线形状保留,仅金额刻度被遮蔽。

**Acceptance Scenarios**:

1. **Given** 趋势图有 ≥2 个数据点, **When** 渲染, **Then** 相邻点之间为平滑曲线连接,目视无锐利折角。
2. **Given** 趋势图渲染, **When** 检查填充, **Then** 折线下方存在垂直渐变(顶部为支出色 `--danger` 较高不透明度、底部为透明),覆盖到 X 轴。
3. **Given** 全部数据点为 0(空周), **When** 渲染, **Then** 折线贴 X 轴、面积退化为极薄/不可见,图表结构不破坏(对齐 027 空数据纪律)。
4. **Given** 隐私模式开启, **When** 渲染, **Then** 平滑折线与渐变面积形状保留,仅 Y 轴金额刻度与 Tooltip 金额被遮蔽(与 027 FR-008 一致)。
5. **Given** recharts 动画, **When** 首次渲染, **Then** 动画不造成 CLS(图表容器高度与稳态一致,对齐 027 FR-013 / 025 动态加载占位纪律)。

### Edge Cases

- **本日支出跨家庭**:本日支出必须受 `familyId` 隔离(与本月支出同一隔离口径),不允许跨家庭聚合。
- **本日支出与"今天"定义**:以 UTC 日为"今天",与 `getMonthSummary` / 027 既有 UTC 口径一致;不引入本地时区"今天"以避免与既有日期边界产生双口径。
- **趋势图窗口固定为本周(不随月份)**:本周 = 当前 UTC 周一 00:00..周日 23:59:59.999(+1ms exclusive end)。月份导航时趋势图不变化(只 hero/分类/预算等月维度区块变)。窗口在"打开页面/重新查询"时取一次,跨 UTC 周一**不自动刷新**(下次查询才更新到新本周)。
- **本周未来日补零**:今天若在本周中间(如周三),本周周四..周日等未来日按 0 补齐,7 桶固定(与 027 daily 补零纪律一致),确保图表骨架稳定。
- **趋势图数据点 < 2**:本周若只有 1 天有数据(其余 0),折线退化为单点,平滑曲线无意义但不报错;渐变面积按单点高度渲染。
- **趋势图全 0**:折线贴轴,面积近乎不可见,Tooltip 仍可悬停显示 `¥0.00`,不破坏图表骨架。
- **趋势环比徽标移除**:原 027 趋势卡的"较上月 ↑X%"徽标在"本周"窗口下语义不再成立,**移除**;趋势卡标题区只保留标题(可改为"本周支出趋势"),不再显示环比百分比。
- **降级**:本日支出子查询失败 → hero 卡右侧显示降级态(`¥--.--`),不抛错、不阻塞其它区块(对齐 027 SC-008 降级纪律)。
- **趋势图加载态**:沿用 025/027 的 `next/dynamic` + Skeleton 占位(recharts 异步加载),面积/平滑变更不改变加载骨架高度。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 首页 hero 卡 **MUST** 在"本月支出"金额**右侧**并排展示"本日支出"金额,两者共享同一张卡片。**视觉层级为主从**:本月支出保持主大数字(027 FR-001 语义不变),本日支出为本月支出右侧的次级较小数字(足够可读但视觉权重低于主数字)。移动端窄屏下不溢出、不换行错位。
- **FR-002**: "本日支出"** MUST** 等于当前 UTC 日 `familyId` 隔离下所有 `type='expense'` 交易金额之和的绝对值(含退款冲减,与 027 R9 type-driven 口径一致);无交易时显示 `¥0.00`。
- **FR-003**: 系统 **MUST** 支持本日支出加载失败时降级为 `¥--.--`(或等价占位),不阻塞本月支出、趋势图、最近账单等其它首页区块渲染。
- **FR-004**: 首页趋势图时间窗 **MUST** 固定为"本周"(当前 UTC 周一 00:00..周日 23:59:59.999 +1ms exclusive end),共 7 个日桶(周一..周日)。**窗口不随月份导航变化**:切换历史月时趋势图仍显示本周,不显示所选月份的趋势。
- **FR-005**: 趋势图每个日桶 **MUST** 为该 UTC 日 `type='expense'` 净支出(`ABS`),含退款冲减;7 桶连续,缺数据日(含今天之后的本周未来日)补 0。
- **FR-006**: 趋势图折线 **MUST** 为平滑曲线连接(相邻点之间无锐利折角)。
- **FR-007**: 趋势图折线下方 **MUST** 有垂直渐变面积填充:顶部为支出色(`--danger`)较高不透明度、底部为透明,覆盖到 X 轴。
- **FR-008**: 隐私模式开启时,本日支出金额、趋势图 Y 轴刻度、Tooltip 金额 **MUST** 被遮蔽(对齐 027 FR-008),趋势图形状(平滑折线 + 渐变面积)保留。
- **FR-009**: 系统 **MUST** 在趋势图区域**移除** 027 既有的"较上月 ↑X%"环比徽标(本周窗口下环比语义不再成立);趋势卡标题区只保留标题文案。
- **FR-010**: hero 卡与趋势图 **MUST** 复用 027 既有的隐私态(全局 `.privacy-on` CSS + `data-amount` 节点)与加载骨架,不新增独立的隐私/加载机制。
- **FR-011**: 趋势图窗口与"本日支出"的"今天/本周"边界 **MUST** 使用 UTC(周一起算 UTC 周一,今天 = UTC 日),取值时机为"查询/重拉时",跨 UTC 午夜/周一**不自动刷新**(不引入定时器)。

### Key Entities *(include if feature involves data)*

- **本日支出(DayExpense)**: 派生聚合值,非持久化实体。= 当前 UTC 日、`familyId` 隔离、`type='expense'` 交易金额之和的 `ABS`(含退款冲减)。单位:分(cents),与 027 金额口径一致。
- **本周趋势(CurrentWeekTrend)**: 派生聚合序列,非持久化。固定 7 个连续 UTC 日桶(当前周一..周日),每桶 = 该日 `type='expense'` 净支出(`ABS`)。窗口固定为当前自然周(不随月份导航变化);今天之后的本周未来日补 0。单位:分。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户打开首页后,**无需滚动、无需切换页面**,即可在同一屏内同时看到"本日支出"(次级)与"本月支出"(主)两个金额,且视觉层级清晰(主从)。
- **SC-002**: 趋势图在本周 7 天窗口下,任意两个相邻有数据的日期桶之间表现为平滑曲线(目视无折角),且折线下方有从支出色到透明的垂直渐变填充。
- **SC-003**: 首页趋势图数据准备完成(加载态结束)后,布局不发生可见跳动(CLS = 0,对齐 027 FR-013 / 025 动态加载纪律)。
- **SC-004**: 本日支出 / 本周趋势的聚合查询 p95 < 既有 `dashboard.summary` 的 500ms 预算(宪章五);新增查询不使首页关键路径 p95 上升超过可忽略量(因聚合复用同一张 `transaction` 表 + `familyId` 索引)。
- **SC-005**: 隐私模式开启时,本日支出、本月支出、趋势金额刻度三者遮蔽态视觉一致(均显示 `***` 或 `••`),趋势形状(平滑线 + 渐变面积)不受影响。
- **SC-006**: 趋势图窗口固定为本周;切换到历史月份(如上月、上上月)时,趋势图**不变化**(仍显示本周),而 hero 本月支出、分类、预算等月维度区块正确变为历史月数据。
- **SC-007**: 趋势卡不再显示"较上月"环比徽标;标题区只剩标题文案(如"本周支出趋势"),无百分比徽标。

## Assumptions

- **图表库**: 复用已安装的 `recharts ^3.9.2`(026 引入),不新增图表库依赖。平滑曲线通过 recharts 既有的曲线插值能力实现(027 现有 `Line` 已用 `type="monotone"`;本 spec 视情况选择更平滑的插值类型);渐变面积通过 recharts 既有 `<defs><linearGradient/></defs>` + `Area`/`Line` 填充实现。
- **异步加载纪律**: 趋势图继续走 `next/dynamic` + `ssr:false` + Skeleton 占位(025 US2 T030 / 027),平滑 + 面积变更不改变加载骨架高度与 chunk 拆分策略。
- **"今天/本周"口径**: 本日支出与本周趋势窗口均以 **UTC** 为边界(今天 = UTC 日;本周 = UTC 周一..周日),与 `getMonthSummary` / `getUtcMonthRange` / `getUtcWeeksInMonth` / 027 既有 UTC 纪律一致;不引入本地时区,避免双口径。
- **"本周"语义 = 当前自然周(不随月份)**: 本周窗口固定为"今天所在的 UTC 周一..周日",不随首页的年月导航变化。月份导航只影响 hero/分类/预算等月维度区块。这是 Clarification Q2 的明确决策——避免"历史月份的本周"语义歧义,实现也更简单(窗口与所选月份解耦)。
- **本周未来日补零**: 今天若在本周中间(如周三),本周周四..周日等未来日按 0 补齐,7 桶固定(Clarification Q3)。这保证图表骨架稳定,代价是折线在"今天"之后会坠到 0。
- **拉取时刷新,不自动刷新**: 窗口/今天在"查询/重拉时"取值,跨 UTC 午夜/周一**不自动刷新**(Clarification Q5)。不引入定时器/轮询(符合宪章六 YAGNI;单人记账场景跨午夜不刷新极少见,下次查询自然更新)。
- **金额单位**: 全链路使用分(cents),展示层除以 100,与 027 `formatCents` 既有口径一致。
- **复用 027 数据契约**: 本日支出与本周趋势作为 `dashboard.summary` 输出的扩展字段返回(对齐 027 既有契约 `contracts/dashboard-summary.md`),不新建独立 procedure,避免违反宪章二(feature 纵切 + 最小工程边界)。
